# SEC//FOUNDATION — тренажёр знаний по кибербезопасности

Интерактивный сайт с теоретическими тестами по фундаменту кибербезопасности.
Пять блоков, банк вопросов с разбором каждого ответа, прогресс сохраняется в браузере.
Полностью статический — работает офлайн, без сервера и без сборки.

## Блоки

1. Сети — OSI, TCP/IP, DNS, порты, firewall, анализ трафика
2. Операционные системы — Linux CLI, права, процессы, Windows Event Logs
3. Security-фундамент и виды атак — CIA, крипто, IAM, фишинг, malware, SQLi/XSS
4. MITRE ATT&CK — тактики, техники, мышление цепочками атак
5. Инструменты и скриптинг — Wireshark, Nmap, SIEM, Python, Bash

## Файлы

- `index.html` — страница и стили
- `quiz-data.js` — банк вопросов (правь и дополняй здесь)
- `quiz-app.js` — логика: проверка ответов, прогресс, сохранение

## Как открыть локально

Просто открой `index.html` в браузере двойным кликом. Всё уже работает.

## Как выложить на GitHub Pages

1. Создай новый публичный репозиторий на GitHub (например `secfoundation`).
2. Загрузи в него все файлы из этой папки (index.html, quiz-data.js, quiz-app.js, README.md).
   Через веб: кнопка «Add file» → «Upload files» → перетащи файлы → Commit.
3. В репозитории открой Settings → Pages.
4. В разделе «Build and deployment» → Source выбери «Deploy from a branch».
5. Branch: `main`, папка `/ (root)` → Save.
6. Через минуту сайт будет доступен по адресу:
   `https://ТВОЙ-ЛОГИН.github.io/secfoundation/`

Готово — заходи туда после каждого блока и закрепляй.

## Как добавлять свои вопросы

Открой `quiz-data.js`. Каждый вопрос — это объект:

```js
{
  q:"Текст вопроса (можно <code>код</code> внутри)",
  cat:"Короткая категория",
  opts:["Вариант A","Вариант B","Вариант C","Вариант D"],
  a:1,                       // индекс правильного ответа (0 = A, 1 = B, ...)
  exp:"Разбор: почему так. Можно <b>выделять</b>."
}
```

Добавляй новые объекты в массив `questions` нужного блока — сайт подхватит их сам.
Прогресс хранится по id вопроса (например `B1-14`), так что при добавлении
вопросов в конец старый прогресс не ломается.

## Сброс прогресса

Кнопка «сбросить весь прогресс» внизу страницы, либо «пройти блок заново»
в каждом блоке. Данные лежат в localStorage браузера — на сервер ничего не уходит.
